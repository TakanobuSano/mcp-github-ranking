パイプライン(毎日・完全無人)

  1. cron-job.org が毎日 JST 定時に起動
    GitHub Actions の schedule は実行時刻が不安定なため、外部 cron から workflow_dispatch を REST API で叩いて起動します(Fine-grained PAT 使用)。
  2. GitHub Actions がランキングを再集計
    GitHub Search API で MCP / Claude Code 関連リポジトリを最大500件収集し、7日前の CSV と比較して「7日間の Stars 増加数」で急上昇を算出します。
  3. Claude API が日本語解説を生成
    上位リポジトリの README を要約して日本語解説を生成します。解説はプロンプトバージョン付きでキャッシュし、キャッシュヒット時は API を呼びません(コストとレート制御)。応答の usage・response id も記録します。
  4. 3経路へ自動配信
    生成結果をコミットし、以下へ配信して1サイクルが完了します。
    📊 Qiita「人気TOP100」記事を毎日更新 🚀 Qiita「週間トレンドTOP30」記事を毎日更新 💬 Teams へ毎週月曜 Top10 を投稿 📝 README / output を自動コミット

設計上の工夫

  • GitHub Actions の schedule 不安定問題は、外部 cron(cron-job.org)からの起動で回避しています。
  • Claude API の呼び出しはキャッシュ優先・リクエスト間スリープつきで、日次運用でもコストとレート制限を制御しています。
  • PAT・Qiita トークン・Teams Webhook はすべて GitHub Secrets 管理で、リポジトリにもこのページにも認証情報は含まれません。
  • このページ自体も静的ファイル1枚で、稼働状況は閲覧時にブラウザから GitHub の公開 API を直接参照しています。

成果物と作り方

  • 成果物: 【毎日更新】人気ランキングTOP100 / 【毎日更新】週間トレンドTOP30

  • 作り方(完全自動化編): cron-job.org で GitHub Actions を毎日確実に起動する
